# AWS S3 + CloudFront デプロイ手順

> 次回セッションで実施予定。このドキュメントを見せながら進めること。

---

## 全体の流れ

```
① S3バケット作成・設定
② ファイルアップロード
③ CloudFrontディストリビューション作成
④ ドメイン取得（任意）
⑤ Route53 + ACM でHTTPS化（ドメインあり時）
⑥ 動作確認
```

---

## ① S3バケット作成・設定

1. AWSコンソール → S3 → 「バケットを作成」
2. バケット名を決める（例: `ts-portfolio`）
3. リージョン: `ap-northeast-1`（東京）
4. 「パブリックアクセスをすべてブロック」→ **オフ**に変更
5. バケットポリシーを設定（後述）
6. 静的ウェブサイトホスティングを有効化
   - インデックスドキュメント: `index.html`
   - エラードキュメント: `index.html`

### バケットポリシー（貼り付け用）
```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicReadGetObject",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::【バケット名】/*"
    }
  ]
}
```
※ `【バケット名】` を実際の名前に変える

---

## ② ファイルアップロード

アップロード対象（`src/` 以下）:
```
index.html
css/style.css
js/i18n.js
js/main.js
js/pixel-icon.js
```

### AWS CLIを使う場合（推奨）
```bash
aws s3 sync ./src s3://【バケット名】 --delete
```

### コンソールから手動でも可
S3バケット → 「アップロード」→ ファイルを選択

---

## ③ CloudFrontディストリビューション作成

1. AWSコンソール → CloudFront → 「ディストリビューションを作成」
2. オリジンドメイン: S3の**静的ウェブサイトエンドポイント**を選択
   - 例: `ts-portfolio.s3-website-ap-northeast-1.amazonaws.com`
   - ※ バケット名そのものではなくエンドポイントURLを選ぶ
3. プロトコル: `HTTPのみ`（S3エンドポイントはHTTPのため）
4. デフォルトルートオブジェクト: `index.html`
5. 「ディストリビューションを作成」

作成後、`xxxx.cloudfront.net` のURLでアクセス可能になる（反映に5〜15分）

## ④ ドメイン取得（任意）

おすすめ取得先:

ドメイン例: `ts-engineer.com` / `ts-dev.jp` など

## ⑤ HTTPS化（ドメインあり時）

### ACM（AWS Certificate Manager）で証明書発行
1. ACM → 「証明書をリクエスト」
2. **リージョンを `us-east-1`（バージニア）に変更** ← 重要
3. ドメイン名を入力 → DNS検証を選択
4. Route53で検証レコードを自動追加

### CloudFrontにドメインを紐付け
1. ディストリビューション → 編集
2. 「代替ドメイン名（CNAME）」に取得したドメインを追加
3. 「カスタムSSL証明書」でACMの証明書を選択

### Route53でDNS設定
1. Route53 → ホストゾーン → レコード作成
2. タイプ: `A`
3. エイリアス: CloudFrontディストリビューションを選択

## ⑥ 動作確認チェックリスト

- [ ] `https://【ドメイン or cloudfront.net】` でページが開く
- [ ] 日英切り替えが動作する
- [ ] ピクセルアイコンが表示される（アクセスカウント増加）
- [ ] スクロールアニメーションが動く
- [ ] モバイルで崩れていない
- [ ] ページ再読み込みで 404 にならない
